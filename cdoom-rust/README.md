# cdoom-rust

Rust workspace for the [strangler-fig migration](https://martinfowler.com/bliki/StranglerFigApplication.html) of Chocolate Doom. Sits beside the vendored C tree at `../chocolate-doom/` and exposes a stable C ABI via `include/cdoom_rust.h` (generated by [cbindgen](https://github.com/mozilla/cbindgen) on build).

## Layout

| Crate | Role |
|---|---|
| `cdoom-core` | Migration modules + `staticlib` linked into Chocolate Doom |
| `cdoom-sys` | Future home for `bindgen` C bindings (placeholder today) |
| `cdoom-verify` | Shared parity checks and baseline test helpers |

## Build

From the repo root:

```bash
./build.sh          # builds Rust (via CMake) + Chocolate Doom
cargo test          # from cdoom-rust/ — Rust unit tests
./scripts/verify-baseline.sh   # timedemo oracle when WAD + binary exist
```

Disable Rust linking:

```bash
cmake .. -DENABLE_CDOOM_RUST=OFF   # in chocolate-doom/build
```

## C ABI (Step 0)

```c
const char *cdoom_rust_version(void);  // static string, do not free
int cdoom_rust_init(void);             // reserved for later phases
```

Query from the game:

```bash
./chocolate-doom/build/src/chocolate-doom -cdoom-rust-info
```

## Migration loop

1. Implement module in `cdoom-core/src/<module>/`
2. Export facade functions in `ffi.rs` → cbindgen updates `cdoom_rust.h`
3. Dual-run behind `#ifdef USE_RUST_<MODULE>` in the matching C file
4. Add parity test in `cdoom-verify`
5. Flip CMake flag; delete C once `./scripts/verify-baseline.sh` passes
